# Быстрая инструкция: Настройка Sandbox для TikTok

## ✅ Да, вы МОЖЕТЕ и ДОЛЖНЫ указать режим песочницы (sandbox)!

**Для новых приложений TikTok ТРЕБУЕТ использование Sandbox environment.**

## Как настроить Sandbox в TikTok Developer Portal:

### Шаг 1: Создайте приложение
1. Войдите в [TikTok for Developers](https://developers.tiktok.com/)
2. Создайте новое приложение или выберите существующее
3. Заполните основную информацию (App name, Description, etc.)

### Шаг 2: Добавьте Sandbox Testers
1. В настройках приложения найдите раздел **"Sandbox"** или **"Add a Sandbox"**
2. Нажмите **"Add Sandbox Tester"**
3. Введите TikTok username тестового пользователя (например: `username` или `@username`)
4. Нажмите "Send Invitation"
5. Повторите для 2-3 тестовых пользователей (рекомендуется)

### Шаг 3: Принятие приглашений
1. Тестовые пользователи должны зайти в свой TikTok аккаунт
2. Найти приглашение в настройках
3. Принять приглашение в sandbox

### Шаг 4: Проверка
- Статус приложения должен показывать **"In Sandbox"**
- Только добавленные тестовые пользователи смогут авторизоваться
- В sandbox можно использовать `http://localhost` для разработки

## Где находится Sandbox в форме?

В форме регистрации приложения:
1. После создания приложения
2. В разделе **"App Management"** -> ваше приложение
3. Найдите вкладку/раздел **"Sandbox"** или **"Add a Sandbox"**
4. Там вы сможете добавить тестовых пользователей

**Примечание:** Sandbox настраивается НЕ в форме регистрации, а ПОСЛЕ создания приложения в настройках приложения.

## Что нужно для ревью?

1. ✅ Приложение создано
2. ✅ Sandbox настроен (добавлены тестовые пользователи)
3. ✅ Тестовые пользователи приняли приглашения
4. ✅ Redirect URI настроен (можно localhost для sandbox)
5. ✅ Demo video записано с sandbox пользователем
6. ✅ Все функции работают в sandbox режиме

## После одобрения:

- Приложение автоматически перейдет в **production режим**
- Все пользователи TikTok смогут авторизоваться
- Sandbox можно оставить или отключить

## Важно:

- **Код приложения не требует изменений** - sandbox настраивается только в Developer Portal
- В sandbox режиме работают те же API endpoints
- Только ограничен доступ - только тестовые пользователи
- После одобрения ограничения снимаются автоматически
